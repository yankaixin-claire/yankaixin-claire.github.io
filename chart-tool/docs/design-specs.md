# 设计规范 — 可编辑图表工具

> 版本：v1.0 | 日期：2026-07-12

---

## 1. 设计理念

### 1.1 核心原则

- **零学习成本**：界面布局直观，用户打开即用，无需教程
- **即时反馈**：所有编辑操作实时反映到图表，无保存按钮
- **克制美学**：淡蓝色调为主，减少视觉噪音，突出数据本身
- **渐进式功能**：常用功能在最显眼位置，高级功能按需展开

### 1.2 适用场景

办公人员快速制作会议用图表、学生完成作业数据可视化、非技术用户探索数据模式。

### 3.2 移动端布局（≤ 768px）

侧边栏移至顶部，变为横向弹性布局。图表区域在下方，保持可滚动。

### 3.3 间距规范

- 卡片内边距：16px / 24px
- 组件间距：20px
- 控件组内间距：6-8px
- 圆角：8px（卡片）、6px（按钮/输入框）

## 4. 组件规范

### 4.1 数据表格

- **单元格**：`contenteditable`，聚焦时蓝色边框 + 浅黄背景
- **表头**：灰色背景 `#f8f9fa`，字号 14px，加粗
- **操作按钮**：圆形 28px，悬浮时高亮
- **添加行**：虚线边框，居中显示 "+ 添加行"
- **最小宽度**：每列 90px，内容超长时横向滚动

### 4.2 图表类型选择器

- 2×2 网格布局，每个按钮含图标 + 文字
- 默认态：灰色边框，白色背景
- 激活态：蓝色边框 + 浅蓝背景 + 蓝色文字
- 切换时有 300ms 动画过渡

### 4.3 文件上传区

- 虚线边框 2px dashed，圆角 6px
- 拖拽悬停时：边框变蓝 + 浅蓝背景
- 支持点击触发文件选择器

### 4.4 Toast 通知

- 固定在屏幕右上角
- 颜色按类型：绿（成功）、红（错误）、橙（警告）、蓝（信息）
- 入场：从右侧滑入，300ms
- 自动消失：2.5 秒后淡出

### 4.5 配色选择器

- 预设方案：彩色圆点，28px，激活时加粗边框 + 外阴影
- 下拉框：标准 select，列出方案名称

## 6. 交互规范

### 6.1 数据编辑流程

```
点击单元格 → 光标进入 → 编辑内容 → 按 Enter / 点击其他单元格 (blur)
→ 自动同步到 Store → 图表实时更新
```

### 6.2 图表切换流程

```
点击类型按钮 → 按钮高亮 → Store.chartType 更新
→ 旧图表 destroy → 新图表 create (300ms 动画)
```

### 6.3 文件上传流程

```
拖拽/点击 → 文件校验 (≤5MB, .csv/.xlsx/.xls)
→ 解析 → Store 更新 → 表格 + 图表同步刷新 → Toast 提示
```

### 6.4 导出流程

```
点击导出 → 按钮变 "⏳ 导出中..." → html2canvas 截图 (2x)
→ 生成 Blob → 触发下载 → Toast "✅ 已导出" → 按钮恢复
```

## 8. 可访问性

- 所有按钮有 `title` 提示
- 输入框有清晰的 `placeholder`
- 聚焦态有蓝色边框 + 外发光 (`box-shadow`)
- 错误状态用红色边框 + Toast 提示
- 操作按钮有明确的图标/文字标签

## 9. 空状态与边界情况

| 场景 | 处理 |
|------|------|
| 无数据 | 图表区显示 "📭 暂无数据，请在表格中添加数据" |
| 单行数据 | 正常渲染（柱状图只有一根柱子） |
| 非数字值 | 自动转为 0，图表不崩溃 |
| 删除最后一列 | 阻止操作（至少保留1列） |
| 删除最后一行 | 阻止操作（至少保留1行） |
| 饼图多系列 | 仅使用第一数据列 |
| 散点图少于3列 | 正常渲染，缺失值填 0 |
| 文件 > 5MB | Toast 错误提示 |
| 不支持的文件格式 | Toast 警告提示 |
| CDN 加载失败 | 页面显示错误提示 |
